使用本地模型运行 Strix，可以实现完全离线、以隐私优先的安全评估。 数据不会离开你的机器，因此非常适合敏感的内部网络或隔离环境。

## 隐私与性能

| 特性 | 本地模型 | 云端模型（GPT-5/Claude 4.5） |
|---------|--------------|--------------------------------|
| **隐私** | 🔒 数据保留在本地 | 数据发送给提供方 |
| **成本** | 免费（仅硬件成本） | 按令牌计费 |
| **推理能力** | 较弱（不擅长智能体协作） | 顶尖水平 |
| **配置** | 较复杂（需要 GPU） | 即开即用 |

<Warning>
**兼容性说明**：Strix 依赖高级智能体能力（工具使用、多步规划、自我纠正）。大多数本地模型，尤其是 70B 参数以下的模型，在这些复杂任务上都比较吃力。

对于关键评估，我们强烈建议使用最先进的云端模型，例如 **Claude 4.5 Sonnet** 或 **GPT-5**。只有在隐私是绝对优先级时才使用本地模型。
</Warning>

## Ollama

[Ollama](https://ollama.ai) 是在 macOS、Linux 和 Windows 上运行本地模型最简单的方式。

### 配置步骤

1. 从 [ollama.ai](https://ollama.ai) 安装 Ollama
2. 拉取一个高性能模型：
   ```bash
   ollama pull qwen3-vl
   ```
3. 配置 Strix：
   ```bash
   export STRIX_LLM="ollama/qwen3-vl"
   export LLM_API_BASE="http://localhost:11434"
   ```

### 推荐模型

我们推荐以下模型，以获得推理能力和工具使用能力之间的最佳平衡：

**推荐模型：**
- **Qwen3 VL** (`ollama pull qwen3-vl`)
- **DeepSeek V3.1** (`ollama pull deepseek-v3.1`)
- **Devstral 2** (`ollama pull devstral-2`)

## LM Studio / OpenAI 兼容接口

如果你使用 LM Studio、vLLM 或其他运行器：

```bash
export STRIX_LLM="openai/local-model"
export LLM_API_BASE="http://localhost:1234/v1"  # 根据需要调整端口
```
