## 开发环境

### 先决条件

- Python 3.12+
- Docker (running)
- [uv](https://docs.astral.sh/uv/)
- Git

### 本地开发

<Steps>
  <Step title="克隆仓库">
    ```bash
    git clone https://github.com/usestrix/strix.git
    cd strix
    ```
  </Step>
  <Step title="安装依赖">
    ```bash
    make setup-dev

    # 或者手动：
    uv sync
    uv run pre-commit install
    ```
  </Step>
  <Step title="配置 LLM">
    ```bash
    export STRIX_LLM="openai/gpt-5.4"
    export LLM_API_KEY="your-api-key"
    ```
  </Step>
  <Step title="运行 Strix">
    ```bash
    uv run strix --target https://example.com
    ```
  </Step>
</Steps>

## 贡献技能

技能是增强智能体能力的专门知识包，位于 `strix/skills/`

### 创建技能

1. 选择合适的分类
2. 创建带有 YAML frontmatter 的 `.md` 文件（包含 `name` 和 `description` 字段）
3. 提供实用示例，例如可工作的 payload、命令和测试用例
4. 提供验证方法以确认发现
5. 通过 PR 提交

## Contributing Code

### Pull Request 流程

1. **先创建 issue** — 描述问题或功能
2. **Fork 并创建分支** — 基于 `main` 开发
3. **进行修改** — 遵循现有代码风格
4. **编写测试** — 确保新功能有覆盖
5. **运行检查** — `make check-all` 应通过
6. **提交 PR** — 关联 issue 并提供上下文

### Code Style

- PEP 8 with 100-character line limit
- Type hints for all functions
- Docstrings for public methods
- Small, focused functions
- Meaningful variable names

## 问题反馈

Include:

- Python version and OS
- Strix version (`strix --version`)
- LLM being used
- Full error traceback
- Steps to reproduce
