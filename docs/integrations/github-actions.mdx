---
title: "GitHub Actions"
description: "在每个 Pull Request 上运行 Strix 安全扫描"
---

将 Strix 集成到你的 GitHub 工作流中，在漏洞到达生产环境前将其捕获。

## 基础工作流

```yaml .github/workflows/security.yml
name: Security Scan

on:
  pull_request:

jobs:
  strix-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Install Strix
        run: curl -sSL https://strix.ai/install | bash

      - name: Run Security Scan
        env:
          STRIX_LLM: ${{ secrets.STRIX_LLM }}
          LLM_API_KEY: ${{ secrets.LLM_API_KEY }}
        run: strix -n -t ./ --scan-mode quick
```

## 必需密钥

将这些密钥添加到你的仓库中：

| 密钥 | 说明 |
|--------|-------------|
| `STRIX_LLM` | 模型名称（例如 `openai/gpt-5.4`） |
| `LLM_API_KEY` | LLM 提供方的 API 密钥 |

## 退出码

当发现漏洞时，工作流会失败：

| 代码 | 结果 |
|------|--------|
| 0 | 通过 - 无漏洞 |
| 2 | 失败 - 发现漏洞 |

## CI 扫描模式

| 模式 | 耗时 | 使用场景 |
|------|----------|----------|
| `quick` | 几分钟 | 每个 PR |
| `standard` | 约 30 分钟 | 每晚构建 |
| `deep` | 1-4 小时 | 发布候选版本 |

<Tip>
PR 建议使用 `quick` 模式以保持反馈速度；将 `deep` 扫描安排为每晚运行。
</Tip>

<Note>
对于 `pull_request` 工作流，Strix 会在 CI/无头运行中自动使用变更文件的 diff-scope。如果 diff 解析失败，请确保拉取完整历史（`fetch-depth: 0`），或者显式设置 `--diff-base`。
</Note>
