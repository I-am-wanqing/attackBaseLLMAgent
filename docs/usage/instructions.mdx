---
title: "自定义指令"
description: "用自定义测试指令引导 Strix"
---

使用指令为扫描提供上下文、凭据或关注重点。

## 行内指令

```bash
strix --target https://app.com --instruction "聚焦认证漏洞"
```

## 基于文件的指令

对于复杂指令，使用文件更合适：

```bash
strix --target https://app.com --instruction-file ./pentest-instructions.md
```

## 常见场景

### 认证测试

```bash
strix --target https://app.com \
  --instruction "使用邮箱 test@example.com 和密码 TestPass123 登录"
```

### 聚焦范围

```bash
strix --target https://api.example.com \
  --instruction "聚焦 /api/users 端点中的 IDOR 漏洞"
```

### 排除项

```bash
strix --target https://app.com \
  --instruction "不要测试 /admin 或 /internal 端点"
```

### API 测试

```bash
strix --target https://api.example.com \
  --instruction "使用 API key 请求头：X-API-Key: abc123。重点关注限流绕过。"
```

## 指令文件示例

```markdown instructions.md
# 渗透测试说明

## 凭据
- 管理员：admin@example.com / AdminPass123
- 用户：user@example.com / UserPass123

## 关注重点
1. 用户资料端点中的 IDOR
2. 不同角色之间的权限提升
3. JWT 令牌篡改

## 不在范围内
- /health 端点
- 第三方集成
```

<Tip>
请尽量具体。好的指令能帮助 Strix 优先处理最有价值的攻击路径。
</Tip>
