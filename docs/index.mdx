Strix 是自主 AI 智能体，像真实黑客一样工作：动态运行你的代码、发现漏洞，并用概念验证进行验证。 它面向需要快速、准确安全测试的开发者和安全团队，避免了人工渗透测试的额外成本以及静态分析工具带来的误报。

## 使用场景

- **应用安全测试** — 在你的应用中检测并验证关键漏洞
- **快速渗透测试** — 几小时内完成渗透测试，而不是几周
- **漏洞赏金自动化** — 自动化研究并生成概念验证，加快报告速度
- **CI/CD 集成** — 在漏洞进入生产环境前将其阻止

## 核心能力

- **完整攻击者工具箱** — 浏览器自动化、HTTP 代理、终端、Python 运行时
- **真实验证** — 概念验证，而非误报
- **多智能体编排** — 专门智能体协同处理复杂目标
- **以开发者为中心的 CLI** — 用于自动化的交互式 TUI 或无头模式

## 安全工具

Strix 智能体自带一整套综合工具箱：

| 工具 | 用途 |
|------|---------|
| HTTP 代理 | 完整的请求/响应操控与分析 |
| 浏览器自动化 | 支持多标签页，适合 XSS、CSRF 和认证流程测试 |
| 终端 | 用于命令执行的交互式 Shell |
| Python 运行时 | 自定义利用代码开发与验证 |
| 侦察 | 自动化 OSINT 和攻击面映射 |
| 代码分析 | 静态与动态分析能力 |

## 漏洞覆盖范围

| 类别 | 示例 |
|----------|----------|
| 访问控制 | IDOR、权限提升、认证绕过 |
| 注入 | SQL、NoSQL、命令注入 |
| 服务端 | SSRF、XXE、反序列化 |
| 客户端 | XSS、原型污染、DOM 漏洞 |
| 业务逻辑 | 竞态条件、工作流操控 |
| 认证 | JWT 漏洞、会话管理 |
| 基础设施 | 配置错误、暴露服务 |

## 多智能体架构

Strix 通过专门智能体的图结构来完成全面安全测试：

- **分布式工作流** — 针对不同攻击面和资产分工的专门智能体
- **可扩展测试** — 并行执行，快速覆盖更多场景
- **动态协同** — 智能体之间共享发现并协同推进

## 快速示例

```bash
# 安装
curl -sSL https://strix.ai/install | bash

# 配置
export STRIX_LLM="openai/gpt-5.4"
export LLM_API_KEY="your-api-key"

# 扫描
strix --target ./your-app
```
