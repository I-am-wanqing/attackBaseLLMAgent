技能是结构化知识包，能够为 Strix 智能体提供特定漏洞类型、技术和测试方法上的深度专业知识。

## 核心理念

LLM 的安全知识范围广，但深度有限。它们知道 _关于_ SQL 注入，却缺少经验丰富的渗透测试人员会使用的细腻技巧，例如解析器怪癖、绕过方法、验证技巧和链式攻击。

技能会把这些深度、专门的知识直接注入到智能体上下文中，把它从通才转变成当前任务的专家。

## 工作方式

当 Strix 为某个具体任务创建智能体时，它会根据上下文最多选择 5 个相关技能：

```python
# 为 JWT 测试创建的智能体会自动加载相关技能
create_agent(
    task="Test authentication mechanisms",
    skills=["authentication_jwt", "business_logic"]
)
```

这些技能会被注入到智能体的系统提示词中，让它可以使用：

- **高级技巧** — 超出标准测试的非显而易见方法
- **可用载荷** — 带有变体的实用示例
- **验证方法** — 如何确认发现并避免误报

## 技能分类

### 漏洞

带有深入利用技巧的核心漏洞类别。

| 技能 | 覆盖范围 |
| ------------------------------------- | ------------------------------------------------------ |
| `authentication_jwt`                  | JWT 攻击、算法混淆、声明篡改 |
| `idor`                                | 对象引用攻击、水平/垂直越权 |
| `sql_injection`                       | SQL 注入变体、WAF 绕过、盲注技巧 |
| `xss`                                 | XSS 类型、过滤器绕过、DOM 利用 |
| `ssrf`                                | 服务器端请求伪造、协议处理器 |
| `csrf`                                | 跨站请求伪造、令牌绕过 |
| `xxe`                                 | XML 外部实体、带外数据外传 |
| `rce`                                 | 远程代码执行向量 |
| `business_logic`                      | 逻辑缺陷、状态操控、竞态条件 |
| `race_conditions`                     | TOCTOU、并发请求攻击 |
| `path_traversal_lfi_rfi`              | 文件包含、路径穿越 |
| `open_redirect`                       | 重定向绕过、URL 解析技巧 |
| `mass_assignment`                     | 属性注入、隐藏参数污染 |
| `insecure_file_uploads`               | 上传绕过、扩展名技巧 |
| `information_disclosure`              | 数据泄露、基于错误的枚举 |
| `subdomain_takeover`                  | 悬空 DNS、云资源接管 |
| `broken_function_level_authorization` | 权限提升、角色绕过 |

### 框架

框架特定的测试模式。

| 技能 | 覆盖范围 |
| --------- | -------------------------------------------- |
| `fastapi` | FastAPI 安全模式、Pydantic 绕过 |
| `nextjs`  | Next.js SSR/SSG 问题、API 路由安全 |

### 技术

第三方服务和平台安全。

| 技能 | 覆盖范围 |
| -------------------- | ---------------------------------- |
| `supabase`           | Supabase RLS 绕过、认证问题 |
| `firebase_firestore` | Firestore 规则、Firebase 认证 |

### 协议

协议特定的测试技术。

| 技能 | 覆盖范围 |
| --------- | ------------------------------------------------ |
| `graphql` | GraphQL 内省、批处理、解析器问题 |

### 工具

用于核心侦察和扫描工具的沙箱 CLI 作业手册。

| 技能 | 覆盖范围 |
| ----------- | ------------------------------------------------------- |
| `nmap`      | 端口/服务扫描语法与高信号扫描模式 |
| `nuclei`    | 模板选择、严重性过滤与速率调优 |
| `httpx`     | HTTP 探测与指纹输出模式 |
| `ffuf`      | 词表 fuzz、匹配器/过滤器策略、递归 |
| `subfinder` | 被动子域枚举与来源控制 |
| `naabu`     | 具备显式速率/验证控制的快速端口扫描 |
| `katana`    | 爬取深度、JS、已知文件行为与注意事项 |
| `sqlmap`    | SQLi 的枚举与受控提取流程 |

## 技能结构

每个技能都是一个带有 YAML frontmatter 元数据的 Markdown 文件：

```markdown
---
name: skill_name
description: Brief description of the skill's coverage
---

# 技能标题

关于该漏洞或技术的核心洞见。

## 攻击面
此技能覆盖的内容以及需要关注的位置。

## 方法论
逐步测试的方法。

## 技巧
如何发现并利用该漏洞。

## 绕过方法
如何绕过常见防护。

## 验证
如何确认发现并避免误报。
```

## 贡献技能

欢迎社区贡献。请在合适的分类中创建 `.md` 文件，并包含 YAML frontmatter（`name` 和 `description` 字段）。优秀的技能应包含：

1. **真实场景技巧** — 在实践中有效的方法
2. **实用载荷** — 带有变体的可用示例
3. **验证步骤** — 如何在没有误报的情况下确认
4. **上下文感知** — 特定版本/环境行为
